Datensatz erstellen¶

Ich nutze einen kleinen 1D-Datensatz, um die Shapley-Werte klar zu visualisieren:

#1D-Datensatz
X      = np.array([[0.1], [0.4], [0.6], [0.9]])
y      = np.array([0,     0,     1,     1   ])
x_test = np.array([[0.5]])
y_test = np.array([1])

1. Gewichtung im KNN-Shapley¶

Beim Basic Explainer (weights="uniform") zählen die K nächsten Nachbarn jeweils gleich stark: $$ w_i = 1 $$

Im Weighted Explainer (weights="distance") fließt stattdessen die umgekehrte Distanz ins Gewicht: $$ w_i = \frac{1}{d(x_i, x_{\mathrm{test}}) + \varepsilon} $$

Dabei sorgt ein kleiner Epsilon-Term dafür, dass die Division durch Null vermieden wird.

1️ Basic KNN-Shapley¶

Hier verwende ich uniforme Gewichte (weights="uniform") und berechne die exakten Shapley-Werte.

# Zelle 3: Basic Explainer mit explain()
model_b    = KNeighborsClassifier(n_neighbors=2, weights="uniform").fit(X, y)
expl_basic = BasicKNNExplainer(model_b, X, y, class_index=1)

iv_basic   = expl_basic.explain(x_test)   
phi_basic  = iv_basic.values              

print("Basic φ:", phi_basic)

Visualisierung der Basic-Werte:

# Basic Visualisierung
plt.figure(figsize=(6,3))

2️ Weighted KNN-Shapley¶

Nun: weights="distance". Zuerst zeige ich den exakten und dann den approximierten Shapley-Score.

2. Warum Approximation? (Laufzeit vs. Genauigkeit)¶

Die genaue Berechnung aller Shapley-Werte braucht extrem viel Aufwand, weil man alle möglichen Gruppen durchprobieren muss. Bei $N$ Merkmalen sind das $2^N$ Kombinationen. Das funktioniert in der Praxis nur, wenn der Datensatz sehr klein ist.

Die DP-Approximation (dynamic programming) schränkt stattdessen auf Teilmengen der Größe bis (K) (und ggf. nur auf die ersten (M)) ein.

Damit reduziert sich der Aufwand auf etwa

$$ O\bigl(M \cdot K \cdot \binom{N}{K}\bigr) $$

und lässt sich für größere Datensätze oder mehrdimensionale Features effizient einsetzen.
Der Trade-off: etwas Genauigkeit gegen deutlich geringere Laufzeit.

# Zelle 5: Weighted Explainer mit den korrekten Methoden
model_w     = KNeighborsClassifier(n_neighbors=2, weights="distance").fit(X, y)
expl_w      = WeightedKNNExplainer(
    model_w, X, y, class_index=1,
    approx_M=None    # None = exakt; für approx: approx_M=int(np.sqrt(len(X)))
)

# exakte Shapley-Werte
phi_w_exact  = expl_w.compute_exact_by_enumeration(x_test[0])

# approximative Shapley-Werte
phi_w_approx = expl_w._compute_wknn_shapley_dp(x_test[0])

print("Weighted exact φ :", phi_w_exact)
print("Weighted approx φ:", phi_w_approx)

Weighted exact φ : [-0.08333333 -0.58333333  0.41666667  0.25      ]
Weighted approx φ: [-0.25       -0.33333333  0.16666667  0.08333333]

3. Interpretation der Shapley-Werte¶

  • Basic (uniform)

    • Der Punkt bei 0.6 trägt mit ≈ 0.42 am stärksten zur korrekten Klassifikation bei.

    • Die Punkte bei 0.1 und 0.4 haben negative Werte, da sie im Mittel die Entscheidung eher „falsch“ beeinflussen.

  • Weighted exact (distance)

    • Der nächstgelegene Punkt (0.4) erhält nun den stärksten Einfluss, weil er den höchsten Gewichtungs-Wert hat.

    • Weiter entfernte Punkte werden entsprechend abgeschwächt.

  • Weighted approx

    • Die approximative Methode fängt den Trend der exakten Werte ein, unterschätzt aber tendenziell den Betrag.

    • Mit größerem approx_M nähert sich die Approx-Variante immer weiter an die exakte Verteilung an, auf Kosten höherer Rechenzeit.

Zusammengefasst zeigt der Vergleich, wie Uniform- vs. Distanz-Gewichtung und Approximation die Verteilung der Shapley-Beiträge verändern und so lokale Einflüsse im KNN-Modell sichtbar werden.

4. Performance-Benchmark¶

-> mittlere Laufzeit der exakten Shapley-Berechnung messen gegenüber der DP-Approximation auf dem 1D-Beispiel, um den Trade-off zwischen Genauigkeit und Performance zu bestimmen.

import timeit
import pandas as pd
import matplotlib.pyplot as plt


def bench_exact():
    expl = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=None)
    expl.compute_exact_by_enumeration(x_test[0])

def bench_approx():
    expl = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=2)
    expl._compute_wknn_shapley_dp(x_test[0])

N = 30
t_ex = timeit.timeit(bench_exact, number=N) / N
t_ap = timeit.timeit(bench_approx, number=N) / N

df = pd.DataFrame({
    "Methode": ["exakt", "approx"],
    "Sekunden pro Lauf": [t_ex, t_ap]
})

plt.figure(figsize=(5,3))
plt.bar(df["Methode"], df["Sekunden pro Lauf"])
plt.title("Durchschnittliche Laufzeit: exact vs. approx")
plt.ylabel("Sekunden pro Lauf")
plt.show()

print(df.to_string(index=False))

  exakt           0.499386
 approx           0.007479

5. Genauigkeitsvergleich: exakte vs. DP-Approximation¶

In diesem Abschnitt vergleiche ich die exakten Shapley-Werte mit denen der DP-Approximation
für verschiedene Werte von approx_M. Die exakte Berechnung hat per Definition einen L₂-Fehler von 0.
Deshalb stelle ich für jede Approximation den Abstand zu den exakten Werten in einem Balkendiagramm dar,
um zu sehen, wie sich die Genauigkeit mit wachsender Stichprobengröße (M) verbessert.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# 1) Exakte Shapley-Werte einmal berechnen
expl_base   = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=None)
exact_vals  = expl_base.compute_exact_by_enumeration(x_test[0])

# 2) Für verschiedene approx_M die DP-Approximation ausführen
Ms      = [1, 2, len(X)]
errors  = []
for M in Ms:
    expl_M      = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=M)
    approx_vals = expl_M._compute_wknn_shapley_dp(x_test[0])
    errors.append(np.linalg.norm(approx_vals - exact_vals))

# 3) DataFrame mit exakter und approx-Werten
df_err = pd.DataFrame({
    "Variante":   ["exakt"] + [f"approx_M={M}" for M in Ms],
    "L2-Fehler":  [0.0]   + errors
})

# 4) Balkendiagramm
plt.figure(figsize=(6,3))
plt.bar(df_err["Variante"], df_err["L2-Fehler"])
plt.title("Genauigkeitsvergleich: exakte vs. DP-Approximation")
plt.ylabel("L2-Fehler gegenüber exakten Werten")
plt.xticks(rotation=30)
plt.ylim(0, df_err["L2-Fehler"].max() * 1.1)
plt.show()

# 5) Tabellarische Ausgabe
print(df_err.to_string(index=False))

     exakt   0.000000
approx_M=1   0.527046
approx_M=2   0.478714
