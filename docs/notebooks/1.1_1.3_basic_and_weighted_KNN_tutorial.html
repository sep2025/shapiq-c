1. Gewichtung im KNN-Shapley¬∂

Beim Basic Explainer (weights="uniform") z√§hlen die K n√§chsten Nachbarn jeweils gleich stark: $\( w_i = 1 \)$

Im Weighted Explainer (weights="distance") flie√üt stattdessen die umgekehrte Distanz ins Gewicht: $\( w_i = \frac{1}{d(x_i, x_{\mathrm{test}}) + \varepsilon} \)$

Dabei sorgt ein kleiner Epsilon-Term daf√ºr, dass die Division durch Null vermieden wird.

2. Warum Approximation? (Laufzeit vs. Genauigkeit)¬∂

Die genaue Berechnung aller Shapley-Werte braucht extrem viel Aufwand, weil man alle m√∂glichen Gruppen durchprobieren muss. Bei \(N\) Merkmalen sind das \(2^N\) Kombinationen. Das funktioniert in der Praxis nur, wenn der Datensatz sehr klein ist.

Die DP-Approximation (dynamic programming) schr√§nkt stattdessen auf Teilmengen der Gr√∂√üe bis (K) (und ggf. nur auf die ersten (M)) ein.

Damit reduziert sich der Aufwand auf etwa

\[ O\bigl(M \cdot K \cdot \binom{N}{K}\bigr) \]

und l√§sst sich f√ºr gr√∂√üere Datens√§tze oder mehrdimensionale Features effizient einsetzen.
Der Trade-off: etwas Genauigkeit gegen deutlich geringere Laufzeit.

3. Interpretation der Shapley-Werte¬∂

  • Basic (uniform)

    • Der Punkt bei 0.6 tr√§gt mit ‚âà 0.42 am st√§rksten zur korrekten Klassifikation bei.

    • Die Punkte bei 0.1 und 0.4 haben negative Werte, da sie im Mittel die Entscheidung eher ‚Äûfalsch‚Äú beeinflussen.

  • Weighted exact (distance)

    • Der n√§chstgelegene Punkt (0.4) erh√§lt nun den st√§rksten Einfluss, weil er den h√∂chsten Gewichtungs-Wert hat.

    • Weiter entfernte Punkte werden entsprechend abgeschw√§cht.

  • Weighted approx

    • Die approximative Methode f√§ngt den Trend der exakten Werte ein, untersch√§tzt aber tendenziell den Betrag.

    • Mit gr√∂√üerem approx_M n√§hert sich die Approx-Variante immer weiter an die exakte Verteilung an, auf Kosten h√∂herer Rechenzeit.

Zusammengefasst zeigt der Vergleich, wie Uniform- vs. Distanz-Gewichtung und Approximation die Verteilung der Shapley-Beitr√§ge ver√§ndern und so lokale Einfl√ºsse im KNN-Modell sichtbar werden.

4. Performance-Benchmark¬∂

-> mittlere Laufzeit der exakten Shapley-Berechnung messen gegen√ºber der DP-Approximation auf dem 1D-Beispiel, um den Trade-off zwischen Genauigkeit und Performance zu bestimmen.

import timeit
import pandas as pd
import matplotlib.pyplot as plt


def bench_exact():
    expl = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=None)
    expl.compute_exact_by_enumeration(x_test[0])


def bench_approx():
    expl = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=2)
    expl._compute_wknn_shapley_dp(x_test[0])


N = 30

t_ap = timeit.timeit(bench_approx, number=N) / N

plt.figure(figsize=(5, 3))
plt.bar(df["Methode"], df["Sekunden pro Lauf"])
plt.title("Durchschnittliche Laufzeit: exact vs. approx")
plt.ylabel("Sekunden pro Lauf")

5. Genauigkeitsvergleich: exakte vs. DP-Approximation¬∂

In diesem Abschnitt vergleiche ich die exakten Shapley-Werte mit denen der DP-Approximation
f√ºr verschiedene Werte von approx_M. Die exakte Berechnung hat per Definition einen L‚ÇÇ-Fehler von 0.
Deshalb stelle ich f√ºr jede Approximation den Abstand zu den exakten Werten in einem Balkendiagramm dar,
um zu sehen, wie sich die Genauigkeit mit wachsender Stichprobengr√∂√üe (M) verbessert.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# 1) Exakte Shapley-Werte einmal berechnen
expl_base = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=None)
exact_vals = expl_base.compute_exact_by_enumeration(x_test[0])

# 2) F√ºr verschiedene approx_M die DP-Approximation ausf√ºhren
Ms = [1, 2, len(X)]
errors = []
for M in Ms:
    expl_M = WeightedKNNExplainer(model_w, X, y, class_index=1, K=2, approx_M=M)
    approx_vals = expl_M._compute_wknn_shapley_dp(x_test[0])
    errors.append(np.linalg.norm(approx_vals - exact_vals))

# 3) DataFrame mit exakter und approx-Werten
df_err = pd.DataFrame(
    {"Variante": ["exakt"] + [f"approx_M={M}" for M in Ms], "L2-Fehler": [0.0] + errors}
)

# 4) Balkendiagramm
plt.figure(figsize=(6, 3))
plt.bar(df_err["Variante"], df_err["L2-Fehler"])
plt.title("Genauigkeitsvergleich: exakte vs. DP-Approximation")
plt.ylabel("L2-Fehler gegen√ºber exakten Werten")
plt.xticks(rotation=30)
plt.ylim(0, df_err["L2-Fehler"].max() * 1.1)
plt.show()

# 5) Tabellarische Ausgabe
print(df_err.to_string(index=False))
