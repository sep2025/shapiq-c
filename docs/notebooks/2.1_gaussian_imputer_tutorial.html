1. Was ist ein Gaussian Conditional Imputer?Â¶

  • Ziel: Fehlende Feature-Werte realistisch ergÃ¤nzen (= Imputation)

  • Annahme: Die Daten folgen einer multivariaten Normalverteilung

  • Grundidee: Wenn ein Teil der Features bekannt ist â†’ andere aus der bedingten Verteilung berechnen

  • Formeln: basiert auf Gleichung (10) und (11) aus Aas et al. (2021), die die bedingte Erwartung und die bedingte Konvarianzmatrix liefern

2. Warum brauchen wir einen Gaussian Conditional Imputer?Â¶

  • Problem: Viele klassische Methoden (z.B. shap) nehmen UnabhÃ¤ngigkeit zwischen Features an â†’ Fehlende Features werden zufÃ¤llig oder mit Mittelwert ersetzt

  • RealitÃ¤t: Features sind oft korreliert (z.B. Einkommen und Bildung)
    â†’ UnabhÃ¤ngigkeitsannahme verzerrt die Shapley-Werte

  • LÃ¶sung mit Gaussian Conditional Imputer: - Nutzt die bedingte multivariate Normalverteilung - Sampelt realistische Werte, basierend auf den bekannten Features - FÃ¼hrt zu prÃ¤ziseren, konsistenteren Shapley-Werten

3.4 Hauptlogik: __call__()Â¶

Diese Methode berechnet fÃ¼r jede gegebene Koalition (Subset von Features) eine erwartete Modellvorhersage. Dabei wird angenommen, dass manche Features (1) und andere unbekannt (0) sind.

Ablauf:

  1. Bekannte Features (1) bleiben unverÃ¤ndert

  2. Unbekannte Features (0) werden mithilfe der bedingten multivariaten Normalverteilung ergÃ¤nzt

  3. Die vervollstÃ¤ndigten Datenpuntkte werden dem Modell Ã¼bergeben

  4. Das Modell sagt Vorhersagen fÃ¼r alle Stichproben voraus

  5. Der empirische Mittelwert dieser Vorhersagen wird zurÃ¼ckgegeben

-> Damit approximiert diese Methode die Shapley-Funktionsauswertung \( v(S) = E[f(x) \mid x_s] \), welche exakt auf Gleichung (2) aus Aas et al. (2021) basiert.

def __call__(self, coalitions: np.ndarray) -> np.ndarray:
    results = []

    for coalition in coalitions:
        # 1: Indizes der bekannten (cond_idx) und fehlenden (target_idx) Features ermitteln
        cond_idx = np.where(coalition == 1)[0]
        target_idx = np.setdiff1d(np.arange(self.n_players), cond_idx)

        # SpezialfÃ¤lle: leere Koalition oder alle Features bekannt
        if len(cond_idx) == 0 or len(target_idx) == 0:
            results.append(self.empty_prediction)
            continue

        # 2: Werte der bekannten Features extrahieren
        cond_values = self.x[0, cond_idx]

        # 3: Bedingte Samples aus der multivariaten Normalverteilung ziehen
        samples, _, _ = self.sample_conditional_gaussian(
            self.data,
            cond_idx,
            cond_values,
            n_samples=self.sample_size,
            random_state=self.random_state,
        )

        # 4: Vorhersage mit dem Modell fÃ¼r alle Samples
        prediction = self.predict(samples)
        mean_pred = np.mean(prediction, axis=0)

        # 5: Mittelwert der Vorhersagen speichern
        results.append(mean_pred)

    # 6: Ergebnisse als Numpy-Array zurÃ¼ckgeben
    return np.array(results)

3.5 Der Kern: sample_conditional_gaussian()Â¶

In dieser Methode wird die zentrale mathematische Idee des Gaussian Imputers umgesetzt. Wir berechnen die bedingte Verteilung der unbekannten Features \(x_{ \bar {S}}\), gegeben die bekannten Features \(x_{{S}}\). Ziel ist es, Stichproben fÃ¼r die fehlenden Features zu generieren.

â–¶ Schritt 1: Erwartungswert und Kovarianz der Gesamtdaten berechnenÂ¶

Wir gehen davon aus, dass alle Daten aus einer multivariaten Normalverteilung stammen. Daher berechnen wir:

  • den Erwartungswert \( \mu \)

  • die Kovarianzmatrix \( \Sigma \)

â–¶ Schritt 2: Aufteilen in bekannte (\(S\)) und unbekannte (\(\bar{S}\)) FeaturesÂ¶

Indizes werden entsprechend getrennt. Danach teilen wir auch den Erwartungswert \( \mu \) und die Kovarianzmatrix \( \Sigma \) in Teilmatrizen auf:

  • \( \mu_S \), \( \mu_{\bar{S}}\)

  • \( \Sigma_{SS} \), \( \Sigma_{\bar{S}\bar{S}} \), \( \Sigma_{\bar{S}S} \), \( \Sigma_{S\bar{S}} \)

â–¶ Schritt 3: Berechnung der bedingten VerteilungÂ¶

Die Formeln stammen aus Aas et al. (2021):

  • Gleichung (10): Bedingter Erwartungswert \( \mu_{\bar{S} \mid S} = \mu_{\bar{S}} + \Sigma_{\bar{S}S} \Sigma_{SS}^{-1} (x_S^* - \mu_S) \)

  • Gleichung (11): Bedingte Kovarianzmatrix \( \Sigma_{\bar{S} \mid S} = \Sigma_{\bar{S}\bar{S}} - \Sigma_{\bar{S}S} \Sigma_{SS}^{-1} \Sigma_{S\bar{S}} \)

Hinweis: Wir verwenden die Pseudoinserse (np.linalg.pinv), falls die Matrix nicht invertierbar ist.

â–¶ Schritt 4: Ziehen von StichprobenÂ¶

Nun erzeugen wir Stichproben aus \( N(\mu_{\bar{S}|S}, \Sigma_{\bar{S}|S}) \) Diese Samples reprÃ¤sentieren plausible Werte fÃ¼r die fehlenden Features.

â–¶ Ergebnis:Â¶

  • conditional_samples: Die gesampelten Werte der fehlenden Features

  • mu_cond: Der berechnete Erwartungswert der bedingten Verteilung

  • sigma_cond: Die zugehÃ¶rige Kovarianzmatrix

def sample_conditional_gaussian(
    data: np.ndarray,
    cond_idx: np.ndarray,
    cond_values: np.ndarray,
    n_samples: int = 1,
    random_state: int | None = None,
) -> tuple[np.ndarray, np.ndarray, np.ndarray]:
    data = np.asanyarray(data)
    cond_values = np.asanyarray(cond_values)

    # 1: Erwartungswert und Kovarianzmatrix
    mu = np.mean(data, axis=0)
    sigma_full = np.cov(data, rowvar=False)

    # 2: Aufteilen in bekannte (S) und unbekannte (SÌ„) Features
    feature_indices = np.arange(data.shape[1])
    target_idx = np.setdiff1d(feature_indices, cond_idx, assume_unique=True)

    mu_s = np.take(mu, cond_idx)  # Î¼_S
    mu_sbar = np.take(mu, target_idx)  # Î¼_{SÌ„}

    # Teilmatrizen aus der Kovarianzmatrix extrahieren
    sigma_ss = sigma_full[np.ix_(cond_idx, cond_idx)]  # Î£_{SS}
    sigma_sbar_sbar = sigma_full[np.ix_(target_idx, target_idx)]  # Î£_{SÌ„SÌ„}
    sigma_sbar_s = sigma_full[np.ix_(target_idx, cond_idx)]  # Î£_{SÌ„S}
    sigma_s_sbar = sigma_sbar_s.T  # Î£_{SSÌ„}

    # 3: Berechnung der bedingten Verteilung
    delta = cond_values - mu_s
    inv_sigma_ss = np.linalg.pinv(sigma_ss)
    mu_cond = mu_sbar + sigma_sbar_s @ inv_sigma_ss @ delta
    sigma_cond = sigma_sbar_sbar - sigma_sbar_s @ inv_sigma_ss @ sigma_s_sbar

    # 4: Ziehen von Stichproben aus der bedingten multivariaten Normalverteilung
    conditional_samples = multivariate_normal(
        mean=mu_cond, cov=sigma_cond, allow_singular=True
    ).rvs(size=n_samples, random_state=random_state)

    # Sicherstellen, dass die Ausgabe 2D ist
    if conditional_samples.ndim == 1:
        conditional_samples = conditional_samples.reshape(1, -1)

    return conditional_samples, mu_cond, sigma_cond

5. Grenzen und VerbesserungsmÃ¶glichkeitenÂ¶

Obwohl der Gaussian Conditional Imputer bei der Apporoximation von bedingten Verteilungen nÃ¼tzlich ist, hat er einige wichtige EinschrÃ¤nkungen:

Diese Annahme ist in realen DatensÃ¤tzen selten erfÃ¼llt, insbesondere wenn die Verteilungen schief oder komplex sind.

Fehlende FlexibilitÃ¤t bei nicht-linearen AbhÃ¤ngigkeiten

Wenn die AbhÃ¤ngigkeiten nicht-linear oder asymmetrisch sind, ist das Modell nicht realistisch.

Um diese EinschrÃ¤nkungen zu Ã¼berwinden, schlÃ¤gt Aas et al. (2021) vor, die Feature-AbhÃ¤ngigkeiten explizit zu modellieren, ohne sich auf Normalverteilung zu verlassen. Der Gaussian Conditional Imputer bietet uns ein einfaches und klares Paradigma, um fehlende Werte unter Annahme einer Normalverteilung zu ergÃ¤nzen. In der Praxis sehen wir jedoch oft nicht-lineare AbhÃ¤ngigkeiten oder asymmetrische Verteilungen, die der Gaussian Imputer nicht gut abbilden kann.
